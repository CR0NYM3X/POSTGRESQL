1. ¿Qué es ADK?

ADK (Agent Development Kit) es un framework de código abierto desarrollado por Google para construir agentes de inteligencia artificial con Python. Un agente de IA es un programa que puede:

  • Entender lenguaje natural: Leer y comprender lo que el usuario escribe.
  • Razonar: Pensar en cómo resolver un problema paso a paso.
  • Actuar: Ejecutar acciones concretas (llamar APIs, calcular cosas, buscar información).
  • Responder: Comunicar los resultados al usuario de forma coherente.

Piensa en un agente como un asistente virtual muy inteligente que no solo habla contigo, sino que también puede hacer cosas reales: buscar en internet, procesar datos, enviar correos, aprobar solicitudes, etc.

¿Por qué usar ADK en lugar de hacer todo desde cero?

Construir un agente de IA desde cero implica resolver muchos problemas complejos: gestionar el historial de conversaciones, coordinar múltiples modelos de IA, manejar errores, gestionar memoria entre sesiones, etc. ADK resuelve todos estos problemas por ti, permitiéndote enfocarte en la lógica de negocio de tu agente.

2. Glosario de Conceptos Clave

Esta sección define con claridad cada concepto fundamental de ADK. Léela antes de comenzar a programar — entender estas definiciones hará que todo lo demás tenga sentido inmediatamente.

🧠 Agente (Agent)

¿Qué es? Un Agente es la entidad principal inteligente del sistema. Es como contratar a un empleado muy capaz: le das un nombre, le explicas cuál es su trabajo (instrucciones), le das las herramientas que necesita, y él se encarga del resto.

En términos técnicos: Es un objeto Python que encapsula un modelo de lenguaje (LLM), instrucciones de comportamiento (system prompt) y un conjunto de herramientas que puede invocar para realizar acciones concretas.

Tipos de agentes: - root_agent (Coordinador): El agente principal que recibe los mensajes del usuario. Puede responder directamente o delegar tareas a sub-agentes especializados. - Sub-agente (Especialista): Un agente diseñado para una tarea específica (ej: solo buscar en internet, solo procesar pagos, solo analizar datos).

Analogía del mundo real: Imagina una empresa de atención al cliente. El root_agent es el recepcionista que atiende a los clientes. Cuando alguien necesita soporte técnico, lo transfiere al técnico especializado (sub-agente). Cuando necesita una factura, lo transfiere al departamento de contabilidad (otro sub-agente).

from google.adk import Agent

# Agente coordinador (recibe al usuario)
root_agent = Agent(
    name="recepcionista",
    model="gemini-2.5-flash",
    description="Coordinador principal que atiende consultas.",
    instruction="Atiende al cliente y delega según el tipo de solicitud.",
    sub_agents=[agente_soporte, agente_ventas],  # Los especialistas
)

⚙️ Runner

¿Qué es? El Runner es el motor de ejecución que hace que todo funcione. Es el componente que: 1. Recibe el mensaje del usuario 2. Lo envía al agente correcto 3. Gestiona las llamadas a herramientas 4. Persiste el historial de conversación 5. Devuelve la respuesta final

Analogía: Si el Agente es el empleado que sabe hacer el trabajo, el Runner es el sistema operativo de la oficina: asegura que los mensajes lleguen, que las acciones se ejecuten, que todo quede registrado y que las respuestas se entreguen correctamente.

Característica clave: El Runner es sin estado — no guarda información en memoria. Delega el almacenamiento a servicios externos como SessionService (historial) y MemoryService (recuerdos a largo plazo).

from google.adk.runners import Runner
from google.adk.sessions import InMemorySessionService

runner = Runner(
    agent=root_agent,           # El agente a ejecutar
    app_name="mi_aplicacion",   # Nombre de la app
    session_service=InMemorySessionService(),  # Dónde guardar el historial
)

🔄 Ciclo Reason-Act (Razonar-Actuar)

¿Qué es? Es el proceso iterativo que sigue el agente cada vez que recibe un mensaje. El agente no simplemente genera texto — piensa, decide si necesita hacer algo, lo hace, analiza el resultado y repite hasta tener una respuesta completa.

El ciclo paso a paso:

1. USUARIO envía mensaje
         ↓
2. AGENTE lee el mensaje y razona con el LLM
   "¿Puedo responder directamente o necesito información adicional?"
         ↓
3. ¿Necesita ejecutar una herramienta?
   ├── NO → Genera la respuesta final → FIN
   └── SÍ → Llama a la herramienta
              ↓
4. HERRAMIENTA ejecuta y devuelve resultado
         ↓
5. AGENTE analiza el resultado y vuelve al paso 2
   (puede llamar más herramientas si es necesario)
         ↓
6. AGENTE genera respuesta final con toda la información
         ↓
7. USUARIO recibe la respuesta

Ejemplo práctico:

Usuario: "¿Cuánto es el 15% de 1,250 dólares y es ese número primo?"

Ciclo 1:
  Razona → "Necesito calcular el 15% de 1250"
  Llama herramienta: calcular_porcentaje(1250, 15) → resultado: 187.5

Ciclo 2:
  Razona → "Ahora necesito verificar si 187 (redondeado) es primo"
  Llama herramienta: verificar_primo(187) → resultado: "187 no es primo (11 × 17)"

Ciclo 3:
  Razona → "Tengo toda la información para responder"
  Genera respuesta: "El 15% de $1,250 es $187.50. El número 187 no es primo,
                     ya que puede dividirse entre 11 y 17."

🔧 Herramientas (Tools)

¿Qué son? Las Herramientas son funciones de Python que el agente puede invocar para realizar acciones concretas en el mundo real. Sin herramientas, el agente solo puede hablar — con herramientas, puede actuar.

¿Por qué son importantes? Los modelos de lenguaje solo pueden generar texto. Las herramientas son el puente entre "hablar sobre hacer algo" y "realmente hacerlo". Por ejemplo: - Sin herramienta: "El precio del euro es aproximadamente 1.08 dólares" (puede ser incorrecto) - Con herramienta: Consulta una API de cambio de divisas y da el precio exacto y actual

🗃️ ToolContext

¿Qué es? El ToolContext es un objeto especial que ADK inyecta automáticamente en las herramientas que lo necesitan. Actúa como una "memoria temporal" dentro de una conversación: permite que diferentes herramientas compartan información y que los datos persistan entre múltiples llamadas durante la misma sesión.

Analogía: Imagina que estás en una reunión y tomas notas en un bloc compartido. Cualquier participante de la reunión puede leer lo que escribiste antes y agregar más información. El ToolContext es ese bloc de notas compartido — pero solo dura mientras dura la "reunión" (la sesión de conversación).

¿Cuándo usarlo? - Cuando una herramienta necesita recordar algo para que otra herramienta lo use después - Para mantener un "estado" de la conversación (ej: items en un carrito de compras) - Para registrar historial de acciones dentro de una sesión

from google.adk.tools.tool_context import ToolContext

def agregar_item(producto: str, cantidad: int, tool_context: ToolContext) -> dict:
    """Agrega un item al pedido actual."""
    # Leer el estado actual del pedido
    pedido = tool_context.state.get("pedido", [])

    # Agregar el nuevo item
    pedido = pedido + [{"producto": producto, "cantidad": cantidad}]

    # Guardar el estado actualizado
    tool_context.state["pedido"] = pedido

    return {"mensaje": f"'{producto}' x{cantidad} agregado al pedido"}


def ver_pedido(tool_context: ToolContext) -> dict:
    """Muestra el pedido actual."""
    # Leer el estado guardado por agregar_item
    pedido = tool_context.state.get("pedido", [])
    return {"items": pedido, "total_items": len(pedido)}

🏢 Sistemas Multi-Agente

¿Qué son? Son arquitecturas donde múltiples agentes trabajan juntos, cada uno especializado en un área, coordinados por un agente principal. En lugar de tener un agente que intente hacer todo (lo cual produce resultados mediocres), divides el trabajo entre especialistas.

¿Cuándo usar multi-agente? - Cuando la tarea es demasiado compleja para un solo agente - Cuando necesitas especialización en diferentes dominios (ej: un agente para análisis financiero, otro para redacción, otro para búsqueda web) - Cuando quieres que diferentes partes del sistema tengan instrucciones, herramientas y modelos distintos - Para mejorar la calidad al reducir el scope de cada agente

Analogía: Una empresa tiene departamentos especializados: Ventas, Soporte Técnico, Contabilidad, RRHH. El recepcionista (root_agent) dirige a cada cliente al departamento correcto. Cada departamento (sub-agente) es experto en su área y no necesita saber lo que hacen los demás.

root_agent (Coordinador / Recepcionista)
├── agente_soporte      → Especialista en soporte técnico
├── agente_ventas       → Especialista en ventas y precios
├── agente_contabilidad → Especialista en facturas y pagos
└── agente_análisis     → Especialista en análisis de datos

👤 Human-in-the-Loop (Aprobación Humana)

¿Qué es? Es un patrón de diseño donde el agente pausa su ejecución y espera la aprobación de un humano antes de continuar con ciertas acciones críticas o irreversibles.

¿Por qué es necesario? Hay acciones que no deberían automatizarse completamente porque son demasiado importantes, costosas o irreversibles: - Transferencias de dinero grandes - Eliminación de datos importantes - Envío de comunicaciones masivas - Cambios en configuraciones críticas de producción

Cómo funciona: 1. El agente detecta que necesita una aprobación 2. Llama a una herramienta de "larga duración" que retorna status: pending 3. El sistema notifica al aprobador humano (por email, Slack, etc.) 4. El humano aprueba o rechaza 5. El agente recibe la decisión y continúa (o se detiene)

Analogía: Un empleado nuevo puede procesar compras pequeñas solo, pero para compras grandes necesita que su jefe firme primero. El Human-in-the-Loop implementa exactamente esta política de aprobaciones.

💾 Memoria a Largo Plazo

¿Qué es? Es la capacidad del agente de recordar información de conversaciones anteriores — no solo de la sesión actual. A diferencia del historial de conversación (que solo dura una sesión), la memoria persiste indefinidamente y puede consultarse en futuras sesiones.

Diferencia entre sesión y memoria: - Sesión (Session): Lo que pasó en la conversación actual. Se pierde cuando cierras el chat. - Memoria (Memory): Lo que el agente "aprendió" sobre ti a lo largo de múltiples conversaciones. Persiste entre sesiones.

Ejemplos de uso: - Un asistente personal que recuerda tus preferencias de alimentación - Un agente de soporte que recuerda los problemas que tuviste en el pasado - Un tutor de idiomas que recuerda en qué temas necesitas más práctica

¿Cómo funciona técnicamente? ADK recupera memorias basándose en relevancia contextual — no trae todo el historial, sino solo lo que es relevante para la conversación actual (similar a cómo funciona la memoria humana).

💻 Ejecución de Código (Code Execution)

¿Qué es? Es la capacidad del agente de generar código Python y ejecutarlo en un entorno seguro y controlado, obteniendo los resultados reales.

¿Por qué es valioso? Permite al agente ir más allá del texto y realmente "hacer los cálculos": - Analizar un archivo CSV con miles de filas - Generar gráficas de visualización de datos - Resolver ecuaciones matemáticas complejas - Procesar y transformar datos

Entorno controlado: El código se ejecuta en un sandbox (contenedor aislado) donde: - Ciertas librerías ya están preimportadas (pandas, numpy, matplotlib, etc.) - Las variables persisten entre ejecuciones dentro de la misma sesión - No se pueden instalar paquetes adicionales - El agente ve los resultados (stdout) del código ejecutado

🔌 Patrón App con Plugins

¿Qué es? El patrón App es una capa de configuración avanzada que envuelve al agente básico y agrega funcionalidad de producción mediante plugins.

¿Cuándo usarlo? Cuando necesitas: - Monitoreo y métricas de uso - Gestión del historial de conversaciones largas (compactación) - Guardado automático de archivos generados - Comportamiento transversal que aplica a todas las interacciones

Plugins disponibles: - ContextFilterPlugin: Mantiene solo las últimas N interacciones para no sobrecargar el contexto del modelo - SaveFilesAsArtifactsPlugin: Guarda automáticamente archivos generados por el agente - Plugins personalizados: Puedes crear los tuyos implementando la clase BasePlugin

📊 Compactación de Eventos

¿Qué es? Es un mecanismo automático que resume el historial de conversación cuando este crece demasiado. Los modelos de lenguaje tienen un límite en cuánto texto pueden procesar de una vez (llamado "ventana de contexto"). La compactación evita que el historial supere ese límite.

¿Cómo funciona? Cuando el historial alcanza cierto tamaño, ADK usa el propio LLM para generar un resumen de las interacciones más antiguas, reemplazándolas por ese resumen. Las interacciones recientes se mantienen intactas.

Analogía: Es como cuando en una reunión muy larga alguien dice: "Para resumir todo lo que discutimos en la última hora: acordamos X, Y y Z. Ahora continuemos con el siguiente punto."

🧪 EvalSet (Evaluación)

¿Qué es? Un EvalSet es un conjunto de casos de prueba para verificar que tu agente se comporta correctamente. Define escenarios específicos, la entrada del usuario, y los criterios que debe cumplir la respuesta del agente.

¿Por qué es importante? Los agentes de IA son no-deterministas — pueden responder diferente a la misma pregunta. La evaluación sistemática te permite: - Detectar regresiones cuando cambias el agente - Medir la calidad del agente objetivamente - Asegurarte de que el agente usa las herramientas correctas

Estructura: Los EvalSets son archivos .evalset.json que contienen casos de prueba con conversaciones de ejemplo y rúbricas de evaluación.

🔀 Patrones de Coordinación Multi-Agente

¿Qué son? Son las estrategias que puede usar el agente coordinador para distribuir trabajo entre sus sub-agentes:

1. Delegación Directa: El rootagent transfiere completamente el control al sub-agente. El sub-agente responde directamente al usuario. El rootagent no interviene en la respuesta.

Usuario → root_agent → [delega] → agente_especialista → Usuario

2. Pipeline (Cadena): La salida de un agente es la entrada del siguiente. El root_agent orquesta la secuencia.

Usuario → root_agent → agente_A → root_agent → agente_B → root_agent → Usuario

3. Paralelo: Múltiples agentes trabajan simultáneamente en partes del problema. El root_agent combina los resultados.

                    → agente_A ↘
Usuario → root_agent              → root_agent (combina) → Usuario
                    → agente_B ↗

3. Instalación y Configuración

Antes de instalar ADK, necesitas: - Python 3.9 o superior — ADK usa características modernas de Python - pip — El gestor de paquetes de Python - Una API key de Google AI o una cuenta de Google Cloud con Vertex AI habilitado

async def ejecutar_conversacion():
    """Simula una conversación completa de múltiples turnos.

    Demuestra cómo el Runner mantiene el estado entre turnos
    y cómo procesar los eventos generados.
    """

    # PASO 1: Crear el servicio de sesiones
    # InMemorySessionService: guarda todo en RAM (para desarrollo)
    # En producción usarías DatabaseSessionService o VertexAiSessionService
    servicio_sesion = InMemorySessionService()

    # PASO 2: Crear el Runner
    runner = Runner(
        agent=agente,
        app_name="demo_runner",          # Nombre de tu aplicación
        session_service=servicio_sesion,
    )

    # PASO 3: Definir los identificadores de la sesión
    # Estos IDs permiten que el Runner encuentre el historial correcto
    config_sesion = {
        "app_name": "demo_runner",
        "user_id": "usuario_001",      # ID único del usuario
        "session_id": "sesion_001",    # ID único de esta conversación
    }

    # PASO 4: Simular múltiples turnos de conversación
    mensajes = [
        "Hola, me llamo Carlos",
        "¿Qué hora es?",
        "¿Recuerdas cómo me llamo?",
    ]

    print("=" * 60)
    print("SIMULACIÓN DE CONVERSACIÓN CON EL RUNNER")
    print("=" * 60)

    for turno, mensaje_texto in enumerate(mensajes, 1):
        print(f"\n[Turno {turno}] Usuario: {mensaje_texto}")

        # PASO 5: Crear el mensaje del usuario en el formato que ADK espera
        mensaje = types.Content(
            role="user",
            parts=[types.Part(text=mensaje_texto)],
        )

        # PASO 6: Ejecutar el runner y procesar eventos
        # run_async() devuelve un generador asíncrono de eventos
        # Cada evento representa un paso en el ciclo Reason-Act
        respuesta_final = ""

        async for evento in runner.run_async(
            new_message=mensaje,
            **config_sesion,
        ):
            # Los eventos tienen diferentes tipos:
            # - Llamadas a herramientas (function_call)
            # - Respuestas de herramientas (function_response)
            # - Texto del agente (model response)

            if evento.content and evento.content.parts:
                for parte in evento.content.parts:
                    # Detectar llamadas a herramientas
                    if hasattr(parte, "function_call") and parte.function_call:
                        print(f"  → Llamando herramienta: {parte.function_call.name}")

                    # Detectar respuesta de texto del agente
                    elif hasattr(parte, "text") and parte.text:
                        if evento.author == agente.name:
                            respuesta_final = parte.text

        print(f"[Turno {turno}] Agente: {respuesta_final}")

    print("\n" + "=" * 60)
    print("FIN DE LA CONVERSACIÓN")


if __name__ == "__main__":
    asyncio.run(ejecutar_conversacion())

def buscar_producto(
    termino_busqueda: str,
    categoria: Optional[str] = None,
) -> dict:
    """Busca productos en el catálogo de TechStore por nombre o categoría.

    Usa esta herramienta cuando el usuario pregunte sobre productos disponibles,
    quiera encontrar un producto específico, o explore el catálogo por categoría.
    NO uses tu conocimiento previo sobre productos — siempre busca en el catálogo.

    Args:
        termino_busqueda: Palabra o frase para buscar. Puede ser el nombre completo
                          o parcial del producto (ej: "laptop", "auriculares pro").
        categoria: Opcional. Filtra por categoría específica. Valores posibles:
                   "laptops", "audio", "smartphones", "accesorios".

    Returns:
        Diccionario con:
        - "productos_encontrados": Lista de productos que coinciden con la búsqueda.
          Cada producto incluye nombre, precio, disponibilidad y especificaciones.
        - "total_resultados": Número total de productos encontrados.
        - "mensaje": Mensaje descriptivo sobre los resultados.

    Example:
        >>> buscar_producto("laptop")
        {"productos_encontrados": [...], "total_resultados": 2, "mensaje": "..."}

        >>> buscar_producto("audio", categoria="audio")
        {"productos_encontrados": [...], "total_resultados": 1, "mensaje": "..."}
    """
    termino = termino_busqueda.lower()
    resultados = []

    for producto in CATALOGO_PRODUCTOS.values():
        # Filtrar por categoría si se especificó
        if categoria and producto["categoria"] != categoria.lower():
            continue

        # Buscar el término en nombre o categoría
        if (
            termino in producto["nombre"].lower()
            or termino in producto["categoria"].lower()
            or termino in producto["id"].lower()
        ):
            resultados.append({
                "id": producto["id"],
                "nombre": producto["nombre"],
                "precio": f"${producto['precio']:,.2f}",
                "disponibilidad": (
                    f"En stock ({producto['stock']} unidades)"
                    if producto["stock"] > 0
                    else "Agotado temporalmente"
                ),
                "categoria": producto["categoria"],
                "garantia": f"{producto['garantia_meses']} meses",
            })

    if not resultados:
        return {
            "productos_encontrados": [],
            "total_resultados": 0,
            "mensaje": (
                f"No encontramos productos con '{termino_busqueda}'"
                + (f" en la categoría '{categoria}'" if categoria else "")
                + ". Intenta con otros términos."
            ),
        }

    return {
        "productos_encontrados": resultados,
        "total_resultados": len(resultados),
        "mensaje": f"Se encontraron {len(resultados)} producto(s).",
    }

def crear_ticket_soporte(
    asunto: str,
    descripcion: str,
    prioridad: str,
    tool_context: ToolContext,
) -> dict:
    """Crea un ticket de soporte técnico para el usuario.

    Usa esta herramienta cuando el usuario tenga un problema técnico,
    necesite asistencia que no puedes resolver en el chat, o solicite
    atención de un técnico especializado.

    Siempre pide al usuario que describa el problema con detalle antes
    de crear el ticket, para que el equipo de soporte tenga toda la información.

    Args:
        asunto: Título breve del problema (máximo 100 caracteres).
                Ejemplos: "Laptop no enciende", "Error al configurar WiFi".
        descripcion: Descripción detallada del problema, incluyendo:
                     - Qué estaba haciendo cuando ocurrió el problema
                     - Mensajes de error que aparecieron
                     - Qué intentos de solución ya realizó
        prioridad: Nivel de urgencia del problema. Valores aceptados:
                   "baja" (consultas generales),
                   "media" (problemas que dificultan el uso),
                   "alta" (el equipo no funciona en absoluto).
        tool_context: Contexto para guardar el ID del ticket en la sesión.

    Returns:
        Diccionario con:
        - "ticket_id": ID único para hacer seguimiento del ticket
        - "asunto": El asunto del ticket creado
        - "prioridad": La prioridad asignada
        - "tiempo_respuesta_estimado": Cuándo esperamos responder
        - "siguiente_paso": Qué debe hacer el usuario ahora
    """
    # Validación de prioridad
    prioridades_validas = ["baja", "media", "alta"]
    if prioridad.lower() not in prioridades_validas:
        return {
            "error": True,
            "mensaje": (
                f"Prioridad '{prioridad}' no válida. "
                f"Usa una de: {', '.join(prioridades_validas)}"
            ),
        }

    # Generar ID único para el ticket
    timestamp = datetime.now().strftime("%Y%m%d%H%M%S")
    ticket_id = f"TECH-{timestamp}"

    # Tiempos de respuesta según prioridad
    tiempos_respuesta = {
        "alta": "2-4 horas hábiles",
        "media": "1 día hábil",
        "baja": "2-3 días hábiles",
    }

    # Crear el ticket en nuestra "base de datos" simulada
    ticket = {
        "id": ticket_id,
        "asunto": asunto,
        "descripcion": descripcion,
        "prioridad": prioridad.lower(),
        "estado": "abierto",
        "fecha_creacion": datetime.now().isoformat(),
        "tiempo_respuesta": tiempos_respuesta[prioridad.lower()],
    }

    TICKETS_SOPORTE[ticket_id] = ticket

    # Guardar el ID del ticket en el estado de la sesión
    # Esto permite al usuario consultar su ticket más tarde en la misma sesión
    tickets_del_usuario = tool_context.state.get("tickets_creados", [])
    tool_context.state["tickets_creados"] = tickets_del_usuario + [ticket_id]
    tool_context.state["ultimo_ticket"] = ticket_id

    return {
        "ticket_id": ticket_id,
        "asunto": asunto,
        "prioridad": prioridad.lower(),
        "estado": "abierto",
        "tiempo_respuesta_estimado": tiempos_respuesta[prioridad.lower()],
        "siguiente_paso": (
            f"Guarda tu número de ticket: {ticket_id}. "
            "Un técnico especializado se comunicará contigo pronto. "
            "Puedes consultar el estado con consultar_ticket()."
        ),
    }


def consultar_ticket(
    ticket_id: str,
    tool_context: ToolContext,
) -> dict:
    """Consulta el estado actual de un ticket de soporte.

    Usa esta herramienta cuando el usuario quiera saber el estado de un
    ticket que creó anteriormente. Si el usuario no recuerda el ID, revisa
    el estado de la sesión donde guardamos los tickets creados.

    Args:
        ticket_id: El ID único del ticket (formato: TECH-AAAAMMDDHHMMSS).
                   Si el usuario no lo recuerda, usa el ID guardado en sesión.
        tool_context: Contexto para leer tickets de la sesión actual.

    Returns:
        Diccionario con el estado actual del ticket, incluyendo ID,
        asunto, prioridad, estado y fecha de creación.
    """
    # Intentar con el ID exacto
    ticket_buscar = ticket_id.strip().upper()

    # Verificar en la base de datos de tickets
    if ticket_buscar in TICKETS_SOPORTE:
        ticket = TICKETS_SOPORTE[ticket_buscar]
        return {
            "encontrado": True,
            "ticket_id": ticket["id"],
            "asunto": ticket["asunto"],
            "prioridad": ticket["prioridad"],
            "estado": ticket["estado"],
            "fecha_creacion": ticket["fecha_creacion"],
            "mensaje": f"Ticket {ticket['id']}: Estado actual es '{ticket['estado']}'.",
        }

    # Buscar en tickets de la sesión actual si el ID no coincide exactamente
    tickets_sesion = tool_context.state.get("tickets_creados", [])
    if tickets_sesion:
        ultimo_ticket_id = tool_context.state.get("ultimo_ticket")
        if ultimo_ticket_id and ultimo_ticket_id in TICKETS_SOPORTE:
            ticket = TICKETS_SOPORTE[ultimo_ticket_id]
            return {
                "encontrado": True,
                "nota": "Mostrando tu ticket más reciente de esta sesión.",
                "ticket_id": ticket["id"],
                "asunto": ticket["asunto"],
                "prioridad": ticket["prioridad"],
                "estado": ticket["estado"],
                "fecha_creacion": ticket["fecha_creacion"],
            }

    return {
        "encontrado": False,

def calcular_imc(peso_kg: float, altura_m: float) -> dict:
    """Calcula el Índice de Masa Corporal (IMC) y lo interpreta.

    Usa esta herramienta cuando el usuario quiera calcular su IMC o
    evaluar si su peso es saludable para su altura.

    NOTA: Esta herramienta solo proporciona información de referencia.
    Siempre recomienda consultar a un médico para evaluación personalizada.

    Args:
        peso_kg: El peso de la persona en kilogramos. Debe ser positivo.
                 Ejemplo: 70.5 para 70.5 kilogramos.
        altura_m: La altura de la persona en metros. Debe ser positivo.
                  Ejemplo: 1.75 para 175 centímetros (= 1.75 metros).

    Returns:
        Diccionario con:
        - "imc": El valor calculado del IMC (redondeado a 1 decimal)
        - "categoria": La categoría según OMS (ej: "Normal", "Sobrepeso")
        - "rango_normal": El rango de peso saludable para esa altura
        - "interpretacion": Explicación amigable del resultado
        - "advertencia": Recomendación de consultar profesional de salud

    Raises:
        No lanza excepciones. Retorna un error descriptivo en el dict si
        los valores son inválidos.

    Example:
        >>> calcular_imc(70.0, 1.75)
        {
            "imc": 22.9,
            "categoria": "Normal",
            "rango_normal": "56.7kg - 80.1kg",
            "interpretacion": "Tu peso está en el rango saludable.",
            "advertencia": "Consulta a tu médico para evaluación personalizada."
        }
    """
    # Validaciones de entrada
    if peso_kg <= 0:
        return {
            "error": True,
            "mensaje": f"El peso debe ser positivo. Recibí: {peso_kg} kg",
        }

    if altura_m <= 0 or altura_m > 3.0:
        return {
            "error": True,
            "mensaje": (
                f"La altura debe estar entre 0 y 3 metros. "
                f"Recibí: {altura_m} m. ¿Quizás la ingresaste en centímetros?"
            ),
        }

    # Calcular IMC: peso / (altura²)
    imc = peso_kg / (altura_m ** 2)

    # Determinar categoría según criterios de la OMS
    if imc < 18.5:
        categoria = "Bajo peso"
        interpretacion = "Tu peso está por debajo del rango saludable."
    elif imc < 25.0:
        categoria = "Normal"
        interpretacion = "¡Excelente! Tu peso está en el rango saludable."
    elif imc < 30.0:
        categoria = "Sobrepeso"
        interpretacion = "Tu peso está ligeramente por encima del rango saludable."
    elif imc < 35.0:
        categoria = "Obesidad Grado I"
        interpretacion = "Considera hablar con un profesional de salud."
    else:
        categoria = "Obesidad Grado II o superior"
        interpretacion = "Te recomendamos consultar a un médico pronto."

    # Calcular rango de peso saludable para esta altura (IMC 18.5 - 24.9)
    peso_min_saludable = 18.5 * (altura_m ** 2)
    peso_max_saludable = 24.9 * (altura_m ** 2)

    return {
        "imc": round(imc, 1),
        "categoria": categoria,
        "rango_normal": f"{peso_min_saludable:.1f}kg - {peso_max_saludable:.1f}kg",
        "interpretacion": interpretacion,
        "advertencia": (
            "Esta es solo una referencia general. "
            "Consulta a tu médico para una evaluación personalizada."
        ),
    }

def registrar_gasto(
    concepto: str,
    monto: float,
    categoria: str,
    tool_context: ToolContext,
) -> dict:
    """Registra un gasto en el presupuesto mensual del usuario.

    Usa esta herramienta cuando el usuario mencione que gastó dinero en algo
    o quiera llevar un registro de sus gastos. Los gastos se guardan durante
    toda la sesión y se pueden consultar con ver_resumen_gastos().

    Args:
        concepto: Descripción del gasto. Ejemplos: "Almuerzo", "Transporte", "Netflix".
        monto: Cantidad gastada en la moneda local. Debe ser positivo.
        categoria: Tipo de gasto. Valores sugeridos: "alimentacion", "transporte",
                   "entretenimiento", "servicios", "salud", "educacion", "otro".
        tool_context: Contexto de la herramienta. ADK lo inyecta automáticamente.

    Returns:
        Diccionario con confirmación del registro y resumen del total acumulado.
    """
    if monto <= 0:
        return {
            "error": True,
            "mensaje": f"El monto debe ser positivo. Recibí: {monto}",
        }

    # Inicializar el registro de gastos si es la primera vez
    if "gastos" not in tool_context.state:
        tool_context.state["gastos"] = []

    # Crear el registro del gasto
    nuevo_gasto = {
        "id": len(tool_context.state["gastos"]) + 1,
        "concepto": concepto,
        "monto": monto,
        "categoria": categoria.lower(),
        "timestamp": datetime.now().strftime("%H:%M"),
    }

    # Guardar en el estado (siempre reasignar, no usar .append())
    tool_context.state["gastos"] = tool_context.state["gastos"] + [nuevo_gasto]

    # Calcular total acumulado
    total = sum(g["monto"] for g in tool_context.state["gastos"])

    return {
        "gasto_registrado": {
            "concepto": concepto,
            "monto": f"${monto:,.2f}",
            "categoria": categoria,
        },
        "total_gastos_sesion": f"${total:,.2f}",
        "cantidad_gastos": len(tool_context.state["gastos"]),
        "mensaje": f"✅ Gasto registrado. Total acumulado esta sesión: ${total:,.2f}",
    }


def ver_resumen_gastos(tool_context: ToolContext) -> dict:
    """Muestra un resumen de todos los gastos registrados en esta sesión.

    Usa esta herramienta cuando el usuario quiera ver un resumen de sus gastos,
    saber cuánto ha gastado en total o cuánto gastó por categoría.

    Args:
        tool_context: Contexto de la herramienta. ADK lo inyecta automáticamente.

    Returns:
        Diccionario con lista de gastos, total general y desglose por categoría.
    """
    gastos = tool_context.state.get("gastos", [])

    if not gastos:
        return {
            "mensaje": "No hay gastos registrados en esta sesión.",
            "sugerencia": "Usa registrar_gasto() para empezar a llevar el control.",
        }

    # Calcular total general
    total_general = sum(g["monto"] for g in gastos)

    # Agrupar por categoría
    por_categoria = {}
    for gasto in gastos:
        cat = gasto["categoria"]
        if cat not in por_categoria:
            por_categoria[cat] = {"total": 0, "cantidad": 0, "items": []}
        por_categoria[cat]["total"] += gasto["monto"]
        por_categoria[cat]["cantidad"] += 1
        por_categoria[cat]["items"].append(f"{gasto['concepto']}: ${gasto['monto']:,.2f}")

    # Formatear el resumen por categoría
    resumen_categorias = {
        cat: {
            "total": f"${datos['total']:,.2f}",
            "cantidad_gastos": datos["cantidad"],
            "porcentaje": f"{(datos['total']/total_general*100):.1f}%",
        }
        for cat, datos in por_categoria.items()
    }

    return {
        "total_gastos": f"${total_general:,.2f}",
        "cantidad_transacciones": len(gastos),
        "por_categoria": resumen_categorias,
        "detalle_gastos": [
            {
                "id": g["id"],
                "hora": g["timestamp"],
                "concepto": g["concepto"],
                "categoria": g["categoria"],
                "monto": f"${g['monto']:,.2f}",
            }
            for g in gastos
        ],
    }

def ver_menu(categoria: Optional[str] = None) -> dict:
    """Muestra el menú completo del restaurante o filtra por categoría.

    Usa esta herramienta cuando el cliente pida ver el menú, pregunte
    qué opciones hay disponibles, o quiera explorar una sección específica.

    Args:
        categoria: Opcional. Muestra solo una sección del menú.
                   Valores válidos: "entradas", "principales", "postres", "bebidas".
                   Si no se especifica, muestra todo el menú.

    Returns:
        Diccionario con los items del menú, organizados por categoría,
        con nombre, precio y descripción de cada plato disponible.
    """
    if categoria:
        # Filtrar por categoría
        items_filtrados = {
            item_id: item
            for item_id, item in MENU.items()
            if item["categoria"] == categoria.lower() and item["disponible"]
        }

        if not items_filtrados:
            return {
                "error": True,
                "mensaje": (
                    f"La categoría '{categoria}' no existe o no tiene items disponibles. "
                    "Categorías válidas: entradas, principales, postres, bebidas"
                ),
            }

        return {
            "categoria": categoria,
            "items": [
                {
                    "id": item_id,
                    "nombre": item["nombre"],
                    "precio": f"${item['precio']:.2f}",
                    "descripcion": item["descripcion"],
                }
                for item_id, item in items_filtrados.items()
            ],
            "total_items": len(items_filtrados),
        }

    # Mostrar menú completo organizado por categoría
    menu_organizado = {}
    categorias_orden = ["entradas", "principales", "postres", "bebidas"]

    for cat in categorias_orden:
        items_cat = [
            {
                "id": item_id,
                "nombre": item["nombre"],
                "precio": f"${item['precio']:.2f}",
                "descripcion": item["descripcion"],
            }
            for item_id, item in MENU.items()
            if item["categoria"] == cat and item["disponible"]
        ]
        if items_cat:
            menu_organizado[cat.capitalize()] = items_cat

    return {
        "restaurante": "La Buena Mesa",
        "menu": menu_organizado,
        "nota": "Todos los precios incluyen impuestos. Propina no incluida.",
    }


def agregar_al_pedido(
    id_item: str,
    cantidad: int,
    tool_context: ToolContext,
    notas_especiales: Optional[str] = None,
) -> dict:
    """Agrega uno o más items al pedido actual del cliente.

    Usa esta herramienta cuando el cliente diga que quiere pedir algo,
    que desea un plato específico, o que quiera agregar algo a su orden.

    Args:
        id_item: El ID del item en el menú (obtenido con ver_menu()).
                 Ejemplos: "pasta-carbonara", "ensalada-cesar", "tiramisu".
        cantidad: Número de unidades a pedir. Debe ser 1 o más.
        tool_context: Contexto de sesión. ADK lo inyecta automáticamente.
        notas_especiales: Instrucciones adicionales para la cocina.
                          Ejemplos: "sin sal", "término medio", "sin gluten si es posible".

    Returns:
        Confirmación del item agregado con el subtotal actualizado del pedido.
    """
    # Validar que el item existe en el menú
    if id_item not in MENU:
        # Intentar encontrar el item por nombre parcial
        coincidencias = [
            (item_id, item["nombre"])
            for item_id, item in MENU.items()
            if id_item.lower() in item["nombre"].lower()
        ]

        if coincidencias:
            sugerencia = ", ".join(f"'{nombre}' (id: {item_id})" for item_id, nombre in coincidencias)
            return {
                "error": True,
                "mensaje": f"No encontré el item '{id_item}'. ¿Quisiste decir: {sugerencia}?",
            }

        return {
            "error": True,
            "mensaje": (
                f"El item '{id_item}' no existe en el menú. "
                "Usa ver_menu() para ver los IDs correctos."
            ),
        }

    item = MENU[id_item]

    # Verificar disponibilidad
    if not item["disponible"]:
        return {
            "error": True,
            "mensaje": (
                f"Lo sentimos, '{item['nombre']}' está agotado por hoy. "
                "¿Deseas ver otras opciones?"
            ),
        }

    # Validar cantidad
    if cantidad < 1:
        return {
            "error": True,
            "mensaje": "La cantidad debe ser al menos 1.",
        }

    # Inicializar el pedido si no existe
    if "pedido" not in tool_context.state:
        tool_context.state["pedido"] = []
        tool_context.state["pedido_iniciado"] = datetime.now().isoformat()

    # Verificar si el item ya está en el pedido
    pedido_actual = tool_context.state["pedido"]
    item_existente = next(
        (i for i in pedido_actual if i["id"] == id_item),
        None
    )

    if item_existente:
        # Si ya existe, incrementar la cantidad
        nueva_cantidad = item_existente["cantidad"] + cantidad
        pedido_actualizado = [
            {**i, "cantidad": nueva_cantidad, "subtotal": nueva_cantidad * item["precio"]}
            if i["id"] == id_item else i
            for i in pedido_actual
        ]
    else:
        # Si no existe, agregar nuevo item
        nuevo_item = {
            "id": id_item,
            "nombre": item["nombre"],
            "precio_unitario": item["precio"],
            "cantidad": cantidad,
            "subtotal": item["precio"] * cantidad,
            "notas": notas_especiales or "",
        }
        pedido_actualizado = pedido_actual + [nuevo_item]

    # Guardar el pedido actualizado en el estado
    tool_context.state["pedido"] = pedido_actualizado

    # Calcular total del pedido
    total = sum(i["subtotal"] for i in pedido_actualizado)

    return {
        "agregado": {
            "nombre": item["nombre"],
            "cantidad": cantidad,
            "precio_unitario": f"${item['precio']:.2f}",
            "subtotal": f"${item['precio'] * cantidad:.2f}",
            "notas": notas_especiales or "Sin notas especiales",
        },
        "total_pedido": f"${total:.2f}",
        "items_en_pedido": len(pedido_actualizado),
        "mensaje": f"✅ {cantidad}x {item['nombre']} agregado(s) a tu pedido.",
    }


def ver_pedido_actual(tool_context: ToolContext) -> dict:
    """Muestra el resumen completo del pedido actual del cliente.

def confirmar_pedido(tool_context: ToolContext) -> dict:
    """Confirma y envía el pedido a la cocina.

    Usa esta herramienta SOLO cuando el cliente haya revisado su pedido
    y confirme explícitamente que quiere ordenar. Siempre muestra el pedido
    completo antes de llamar esta herramienta para que el cliente pueda
    revisar todo.

    Args:
        tool_context: Contexto de sesión para leer y marcar el pedido como confirmado.

    Returns:
        Número de orden, tiempo estimado de preparación y resumen del pedido confirmado.
    """
    pedido = tool_context.state.get("pedido", [])

    if not pedido:
        return {
            "error": True,
            "mensaje": "No hay items en tu pedido. Usa agregar_al_pedido() primero.",
        }

    # Verificar que el pedido no fue ya confirmado
    if tool_context.state.get("pedido_confirmado"):
        numero_orden = tool_context.state.get("numero_orden")
        return {
            "error": True,
            "mensaje": (
                f"Tu pedido ya fue confirmado (Orden #{numero_orden}). "
                "Para agregar más items, necesitas hacer un nuevo pedido."
            ),
        }

    # Generar número de orden
    numero_orden = f"ORD-{datetime.now().strftime('%H%M%S')}"

    # Estimar tiempo de preparación
    tiene_principales = any(
        MENU.get(i["id"], {}).get("categoria") == "principales"
        for i in pedido
    )
    tiempo_estimado = "25-35 minutos" if tiene_principales else "10-15 minutos"

    # Marcar el pedido como confirmado en el estado
    tool_context.state["pedido_confirmado"] = True
    tool_context.state["numero_orden"] = numero_orden
    tool_context.state["hora_confirmacion"] = datetime.now().strftime("%H:%M")

    total = sum(item["subtotal"] for item in pedido)

    return {
        "confirmado": True,
        "numero_orden": numero_orden,
        "hora_confirmacion": datetime.now().strftime("%H:%M"),
        "tiempo_estimado": tiempo_estimado,
        "resumen_final": {
            "items": [f"{i['cantidad']}x {i['nombre']}" for i in pedido],
            "total": f"${total:.2f}",
        },
        "mensaje": (
            f"¡Perfecto! Tu orden #{numero_orden} fue enviada a la cocina. "
            f"Tiempo estimado: {tiempo_estimado}. ¡Buen provecho! 🍽️"
        ),
    }

"""
Agente Bancario con Flujo de Aprobación Humana.

Implementa un sistema de solicitud de préstamos donde:
- Préstamos hasta $5,000: se aprueban automáticamente
- Préstamos de $5,001 a $50,000: requieren aprobación del analista
- Préstamos mayores a $50,000: requieren aprobación del comité

El patrón Human-in-the-Loop pausa la ejecución del agente hasta
que un humano tome una decisión. Esto es crítico para:
- Acciones financieras irreversibles
- Decisiones que afectan a múltiples partes
- Cumplimiento regulatorio
- Gestión de riesgo

Arquitectura del flujo:
    Cliente → Agente (evaluación inicial)
                 ↓ si requiere aprobación
           Analista/Comité (revisión humana)
                 ↓ aprueba o rechaza
           Agente (continúa con la decisión)
                 ↓
           Cliente (resultado final)
"""

def evaluar_perfil_crediticio(
    nombre_cliente: str,
    ingreso_mensual: float,
    monto_solicitado: float,
    plazo_meses: int,
    tool_context: ToolContext,
) -> dict:
    """Evalúa el perfil crediticio del cliente y calcula viabilidad del préstamo.

    Esta es la primera herramienta a usar cuando un cliente solicita un préstamo.
    Analiza los datos financieros y determina qué nivel de aprobación se requiere.

    Args:
        nombre_cliente: Nombre completo del cliente solicitante.
        ingreso_mensual: Ingreso mensual verificado del cliente en dólares.
                         Debe ser positivo.
        monto_solicitado: Cantidad total del préstamo solicitado en dólares.
                          Mínimo $500, máximo $500,000.
        plazo_meses: Duración del préstamo en meses.
                     Valores aceptados: 6, 12, 24, 36, 48, 60.
        tool_context: Contexto de sesión para guardar datos del solicitante.

    Returns:
        Diccionario con la evaluación crediticia incluyendo:
        - ratio_deuda_ingreso: Porcentaje del ingreso que iría al pago mensual
        - nivel_aprobacion: "automatica", "analista", o "comite"
        - cuota_mensual_estimada: Pago mensual aproximado
        - evaluacion: "aprobado", "pendiente_revision", o "rechazado"
        - razon: Explicación de la evaluación
    """
    # Validaciones básicas
    plazos_validos = [6, 12, 24, 36, 48, 60]
    if plazo_meses not in plazos_validos:
        return {
            "error": True,
            "mensaje": (
                f"Plazo {plazo_meses} meses no válido. "
                f"Opciones disponibles: {', '.join(str(p) for p in plazos_validos)} meses."
            ),
        }

    if monto_solicitado < 500:
        return {
            "error": True,
            "mensaje": "El monto mínimo de préstamo es $500.",
        }

    if ingreso_mensual <= 0:
        return {
            "error": True,
            "mensaje": "El ingreso mensual debe ser mayor a 0.",
        }

    # Calcular cuota mensual (tasa de interés simplificada)
    tasa_anual = 0.12  # 12% anual (simplificado)
    tasa_mensual = tasa_anual / 12
    cuota_mensual = (
        monto_solicitado * tasa_mensual * (1 + tasa_mensual) ** plazo_meses
    ) / ((1 + tasa_mensual) ** plazo_meses - 1)

    # Calcular ratio deuda/ingreso
    ratio_di = (cuota_mensual / ingreso_mensual) * 100

    # Determinar nivel de aprobación requerido
    if ratio_di > 60:
        evaluacion = "rechazado"
        nivel_aprobacion = "ninguno"
        razon = f"La cuota mensual (${cuota_mensual:.2f}) representa el {ratio_di:.1f}% de sus ingresos, superando el límite del 60%."
    elif monto_solicitado <= 5000:
        evaluacion = "aprobado_automaticamente"
        nivel_aprobacion = "automatica"
        razon = f"Monto dentro del límite de aprobación automática. Ratio deuda/ingreso: {ratio_di:.1f}%"
    elif monto_solicitado <= 50000:
        evaluacion = "pendiente_revision"
        nivel_aprobacion = "analista"
        razon = f"Monto requiere revisión de analista de crédito. Ratio deuda/ingreso: {ratio_di:.1f}%"
    else:
        evaluacion = "pendiente_revision"
        nivel_aprobacion = "comite"
        razon = f"Monto mayor a $50,000 requiere aprobación del comité. Ratio deuda/ingreso: {ratio_di:.1f}%"

    # Guardar datos de la evaluación en el estado de la sesión
    tool_context.state["solicitud_prestamo"] = {
        "nombre_cliente": nombre_cliente,
        "ingreso_mensual": ingreso_mensual,
        "monto_solicitado": monto_solicitado,
        "plazo_meses": plazo_meses,
        "cuota_mensual": round(cuota_mensual, 2),
        "ratio_di": round(ratio_di, 2),
        "nivel_aprobacion": nivel_aprobacion,
        "evaluacion_inicial": evaluacion,
    }

    return {
        "cliente": nombre_cliente,
        "monto_solicitado": f"${monto_solicitado:,.2f}",
        "plazo": f"{plazo_meses} meses",
        "cuota_mensual_estimada": f"${cuota_mensual:.2f}",
        "ratio_deuda_ingreso": f"{ratio_di:.1f}%",
        "nivel_aprobacion_requerido": nivel_aprobacion,
        "evaluacion": evaluacion,
        "razon": razon,
        "siguiente_paso": (
            "Proceder con aprobar_prestamo_automatico()" if nivel_aprobacion == "automatica" and evaluacion != "rechazado"
            else "Enviar para revisión humana con solicitar_aprobacion_humana()" if "pendiente" in evaluacion
            else "Informar al cliente del rechazo"
        ),
    }


def aprobar_prestamo_automatico(tool_context: ToolContext) -> dict:
    """Aprueba y procesa un préstamo automáticamente (para montos hasta $5,000).

    Solo usa esta herramienta si evaluar_perfil_crediticio() indicó
    que el nivel de aprobación es "automatica". No la uses para préstamos
    que requieran aprobación de analista o comité.

    Args:
        tool_context: Contexto de sesión con los datos de la evaluación previa.

    Returns:
        Confirmación de aprobación con número de contrato y condiciones del préstamo.
    """
    solicitud = tool_context.state.get("solicitud_prestamo")

    if not solicitud:
        return {
            "error": True,
            "mensaje": "No hay solicitud de préstamo activa. Usa evaluar_perfil_crediticio() primero.",
        }

    if solicitud["nivel_aprobacion"] != "automatica":
        return {
            "error": True,
            "mensaje": (
                f"Este préstamo requiere aprobación de: {solicitud['nivel_aprobacion']}. "
                "Usa solicitar_aprobacion_humana() en su lugar."
            ),
        }

    # Generar número de contrato
    numero_contrato = f"PREST-{datetime.now().strftime('%Y%m%d%H%M%S')}"

    # Marcar como aprobado en el estado
    tool_context.state["prestamo_aprobado"] = True
    tool_context.state["numero_contrato"] = numero_contrato

    return {
        "aprobado": True,
        "numero_contrato": numero_contrato,
        "cliente": solicitud["nombre_cliente"],
        "monto_aprobado": f"${solicitud['monto_solicitado']:,.2f}",
        "plazo": f"{solicitud['plazo_meses']} meses",
        "cuota_mensual": f"${solicitud['cuota_mensual']:.2f}",
        "fecha_aprobacion": datetime.now().strftime("%d/%m/%Y %H:%M"),
        "siguiente_paso": (
            "El desembolso se realizará en 24-48 horas hábiles. "
            "Se enviará el contrato al correo registrado."
        ),
    }

def solicitar_aprobacion_humana(
    nivel_aprobacion: str,
    tool_context: ToolContext,
) -> dict[str, Any]:
    """Envía la solicitud de préstamo a revisión humana y pausa la ejecución.

    IMPORTANTE: Esta es una herramienta de LARGA DURACIÓN. Cuando se llama,
    el agente entra en estado de espera hasta que el analista o comité
    proporcione su decisión. Esto puede tomar horas o días.

    Usa esta herramienta cuando evaluar_perfil_crediticio() indique que
    el nivel de aprobación es "analista" o "comite".

    Args:
        nivel_aprobacion: Quién debe aprobar. Valores: "analista" o "comite".
        tool_context: Contexto de sesión con los datos de la solicitud.

    Returns:
        Diccionario con status "pending" y el ID del ticket de revisión.
        El resultado final llegará cuando el aprobador humano intervenga.
    """
    solicitud = tool_context.state.get("solicitud_prestamo")

    if not solicitud:
        return {
            "status": "error",
            "mensaje": "No hay solicitud activa. Usa evaluar_perfil_crediticio() primero.",
        }

    # Generar ID de ticket de revisión
    ticket_id = f"REV-{datetime.now().strftime('%Y%m%d%H%M%S')}"

    # Guardar el ID del ticket en el estado
    tool_context.state["ticket_revision"] = ticket_id
    tool_context.state["aprobacion_pendiente"] = True

    # En un sistema real, aquí enviarías:
    # - Email al analista/comité con los detalles
    # - Notificación en el sistema de gestión de préstamos
    # - Registro en la base de datos con status "pending"
    print(f"[Sistema] Ticket de revisión {ticket_id} enviado a: {nivel_aprobacion}")
    print(f"[Sistema] Datos de la solicitud: {solicitud}")

    # RETORNAR "pending" ES LA CLAVE DEL HUMAN-IN-THE-LOOP
    # El LongRunningFunctionTool detecta este status y pausa al agente
    return {
        "status": "pending",           # ← Este "pending" pausa la ejecución
        "ticket_id": ticket_id,
        "enviado_a": nivel_aprobacion,
        "datos_solicitud": {
            "cliente": solicitud["nombre_cliente"],
            "monto": f"${solicitud['monto_solicitado']:,.2f}",
            "plazo": f"{solicitud['plazo_meses']} meses",
            "cuota_mensual": f"${solicitud['cuota_mensual']:.2f}",
            "ratio_di": f"{solicitud['ratio_di']:.1f}%",
        },
        "mensaje_cliente": (
            f"Tu solicitud fue enviada a revisión de {nivel_aprobacion}. "
            f"ID de seguimiento: {ticket_id}. "
            "Te notificaremos la decisión en 1-3 días hábiles."
        ),
    }

"""
Asistente Personal con Memoria a Largo Plazo.

Implementa un asistente que recuerda preferencias, historial de conversaciones
y datos del usuario entre sesiones. Demuestra el uso de load_memory_tool
y before_agent_callback para gestionar memoria persistente.

La memoria en ADK funciona así:
1. El agente tiene una conversación con el usuario
2. Al finalizar la sesión, ADK puede guardar memorias relevantes
3. En la próxima sesión, el agente puede recuperar memorias relevantes
4. El agente usa esas memorias para personalizar las respuestas

NOTA: Para que la memoria persista realmente entre sesiones,
necesitas configurar un MemoryService persistente (Vertex AI Memory Bank
o similar). InMemoryMemoryService solo funciona dentro del mismo proceso.

def guardar_preferencia_usuario(
    categoria: str,
    preferencia: str,
    tool_context: ToolContext,
) -> dict:
    """Guarda una preferencia o dato importante del usuario en la sesión.

    Usa esta herramienta cuando el usuario comparta información sobre:
    - Sus preferencias (ej: "prefiero respuestas cortas")
    - Sus datos personales (ej: "trabajo en marketing")
    - Sus objetivos (ej: "quiero aprender Python")
    - Sus limitaciones (ej: "soy principiante en tecnología")

    Esta información se guardará en el estado de la sesión para
    personalizar las respuestas durante toda la conversación.

    Args:
        categoria: Tipo de información. Ejemplos: "preferencia_respuesta",
                   "ocupacion", "objetivo", "nivel_tecnico", "nombre".
        preferencia: El valor o dato a recordar.
        tool_context: Contexto de sesión para persistir la información.

    Returns:
        Confirmación de que la preferencia fue guardada.
    """
    # Inicializar el perfil del usuario si no existe
    if "perfil_usuario" not in tool_context.state:
        tool_context.state["perfil_usuario"] = {}

    # Guardar la preferencia
    tool_context.state["perfil_usuario"][categoria] = {
        "valor": preferencia,
        "guardado_en": datetime.now().strftime("%d/%m/%Y %H:%M"),
    }

    return {
        "guardado": True,
        "categoria": categoria,
        "valor": preferencia,
        "mensaje": f"✅ Recordaré que tu {categoria} es: {preferencia}",
    }


def ver_perfil_guardado(tool_context: ToolContext) -> dict:
    """Muestra toda la información guardada sobre el usuario en esta sesión.

    Usa esta herramienta cuando el usuario pregunte qué recuerdas de él,
    cuando necesites verificar las preferencias guardadas, o cuando quieras
    personalizar una respuesta basándote en el perfil conocido.

    Args:
        tool_context: Contexto de sesión con el perfil del usuario.

    Returns:
        Perfil completo del usuario con toda la información guardada en la sesión.
    """
    perfil = tool_context.state.get("perfil_usuario", {})

    if not perfil:
        return {
            "mensaje": "No tengo información guardada sobre ti en esta sesión.",
            "sugerencia": "Cuéntame sobre ti y lo recordaré para personalizar mi ayuda.",
        }

    return {
        "perfil_conocido": {
            cat: datos["valor"]
            for cat, datos in perfil.items()
        },
        "items_recordados": len(perfil),
        "mensaje": "Esto es lo que sé sobre ti en esta sesión.",
    }

class MetricasPlugin(BasePlugin):
    """Plugin de monitoreo que registra métricas detalladas de uso.

    Implementa hooks del ciclo de vida para capturar:
    - Número de invocaciones del agente
    - Número de llamadas al modelo LLM
    - Tiempos de respuesta
    - Cualquier métrica personalizada que necesites

    Google Cloud Monitoring, Datadog, New Relic, o tu propio sistema.

    Attributes:
        invocaciones: Contador total de invocaciones del agente.
        llamadas_llm: Contador total de llamadas al modelo LLM.
        tiempo_inicio: Timestamp del inicio del último turno.
        tiempos_respuesta: Lista de tiempos de respuesta en segundos.
    """

    def __init__(self, nombre_servicio: str = "agente_produccion") -> None:
        """Inicializa el plugin de métricas.

        Args:
            nombre_servicio: Identificador del servicio para los logs.
                             Útil si tienes múltiples agentes y necesitas
                             diferenciarlos en los logs.
        """
        super().__init__(name="metricas_plugin")
        self.nombre_servicio = nombre_servicio
        self.invocaciones: int = 0
        self.llamadas_llm: int = 0
        self.tiempo_inicio: float | None = None
        self.tiempos_respuesta: list[float] = []

    async def before_agent_callback(
        self,
        *,
        agent: BaseAgent,
        callback_context: CallbackContext,
    ) -> None:
        """Hook ejecutado ANTES de cada invocación del agente.

        Args:
            agent: El agente que está por ejecutarse.
            callback_context: Contexto con estado de la sesión.
        """
        import time
        self.invocaciones += 1
        self.tiempo_inicio = time.time()
        print(
            f"[Métricas] Invocación #{self.invocaciones} | "
            f"Agente: {agent.name} | "
            f"Sesión: {callback_context.state.get('session_id', 'desconocida')}"
        )

    async def after_agent_callback(
        self,
        *,
        agent: BaseAgent,
        callback_context: CallbackContext,
    ) -> None:
        """Hook ejecutado DESPUÉS de cada invocación del agente.

        Args:
            agent: El agente que terminó de ejecutarse.
            callback_context: Contexto con estado de la sesión.
        """
        import time
        if self.tiempo_inicio:
            tiempo_respuesta = time.time() - self.tiempo_inicio
            self.tiempos_respuesta.append(tiempo_respuesta)
            print(
                f"[Métricas] Turno completado en {tiempo_respuesta:.2f}s | "
                f"Promedio: {sum(self.tiempos_respuesta)/len(self.tiempos_respuesta):.2f}s"
            )

    async def before_model_callback(
        self,
        *,
        callback_context: CallbackContext,
        llm_request: LlmRequest,
    ) -> None:
        """Hook ejecutado ANTES de cada llamada al modelo LLM.

        Args:
            callback_context: Contexto del callback.
            llm_request: La petición que se enviará al modelo.
        """
        self.llamadas_llm += 1
        # En producción, aquí podrías validar contenido, aplicar
        # filtros adicionales o registrar el costo estimado de la llamada.

    def obtener_reporte(self) -> dict:
        """Genera un reporte de métricas acumuladas.

        Returns:
            Diccionario con todas las métricas registradas desde el inicio.
        """
        promedio_tiempo = (
            sum(self.tiempos_respuesta) / len(self.tiempos_respuesta)
            if self.tiempos_respuesta else 0
        )

        }

# PLUGIN 2: SEGURIDAD Y RATE LIMITING
# ─────────────────────────────────────────────────────────────────────────────


class SeguridadPlugin(BasePlugin):
    """Plugin de seguridad que implementa rate limiting y validaciones.

    Protege el agente de uso abusivo limitando el número de solicitudes
    por sesión y detectando patrones problemáticos.

    Attributes:
        max_invocaciones_por_sesion: Límite de turnos por sesión.
        invocaciones_por_sesion: Diccionario que rastrea uso por sesión.
    """

    def __init__(self, max_invocaciones: int = 50) -> None:
        """Inicializa el plugin de seguridad.

        Args:
            max_invocaciones: Máximo de turnos permitidos por sesión.
        """
        super().__init__(name="seguridad_plugin")
        self.max_invocaciones_por_sesion = max_invocaciones
        self.invocaciones_por_sesion: dict[str, int] = {}

    async def before_agent_callback(
        self,
        *,
        agent: BaseAgent,
        callback_context: CallbackContext,
    ) -> None:
        """Verifica rate limiting antes de procesar cada solicitud.

        Args:
            agent: El agente que está por ejecutarse.
            callback_context: Contexto con información de la sesión.

        Raises:
            RuntimeError: Si la sesión supera el límite de invocaciones.
        """
        session_id = callback_context.state.get("session_id", "default")
        if session_id not in self.invocaciones_por_sesion:
            self.invocaciones_por_sesion[session_id] = 0
        self.invocaciones_por_sesion[session_id] += 1
        count = self.invocaciones_por_sesion[session_id]
        if count > self.max_invocaciones_por_sesion:
            print(f"[Seguridad] ⚠️ Límite de sesión alcanzado: {count} invocaciones")

14. LiteLLM Model Connector

14.1 Descripción General

LiteLLM es una librería Python que actúa como capa de traducción para modelos y servicios de hosting, proporcionando una interfaz estandarizada compatible con OpenAI para más de 100+ LLMs. ADK integra LiteLLM a través de la clase LiteLlm, permitiéndote acceder a proveedores como OpenAI, Anthropic (sin Vertex AI), Cohere y muchos más. También puedes ejecutar modelos open-source localmente o en servidores propios para mayor control, ahorro de costos, privacidad o uso offline.

ModalidadCómo usarlo
Modelo remotoUsa la clase LiteLlm y asígnala como parámetro model de LlmAgent
Modelo localConfigura LiteLlm apuntando a tu servidor local (Ollama, vLLM, etc.)

⚠️ Windows — UnicodeDecodeError con LiteLLM
En Windows, LiteLLM puede intentar leer archivos de caché usando la codificación por defecto (cp1252) en lugar de UTF-8. Para evitarlo, establece la variable de entorno PYTHONUTF8=1.

"""
Script para ejecutar el agente programáticamente.

Útil cuando necesitas integrar el agente en una aplicación existente,
hacer llamadas desde código Python, o crear tests de integración manuales.

Uso:
    python scripts/ejecutar_agente.py
    python scripts/ejecutar_agente.py --mensaje "¿Qué laptops tienen?"
"""

import asyncio
import argparse
from typing import AsyncGenerator

from google.adk.runners import Runner
from google.adk.sessions import InMemorySessionService
from google.genai import types

# Importar el agente (import absoluto en scripts)
from agente_soporte_tech import agent


async def ejecutar_un_turno(
    runner: Runner,
    mensaje_texto: str,
    app_name: str,
    user_id: str,
    session_id: str,
) -> str:
    """Ejecuta un solo turno de conversación y retorna la respuesta final.

    Args:
        runner: El Runner configurado con el agente.
        mensaje_texto: El mensaje del usuario.
        app_name: Nombre de la aplicación.
        user_id: ID del usuario.
        session_id: ID de la sesión de conversación.

    Returns:
        La respuesta final del agente como string.
    """
    mensaje = types.Content(
        role="user",
        parts=[types.Part(text=mensaje_texto)],
    )

    respuesta_final = ""
    herramientas_usadas = []

    async for evento in runner.run_async(
        new_message=mensaje,
        app_name=app_name,
        user_id=user_id,
        session_id=session_id,
    ):
        if not evento.content or not evento.content.parts:
            continue

        for parte in evento.content.parts:
            # Registrar herramientas llamadas
            if hasattr(parte, "function_call") and parte.function_call:
                herramientas_usadas.append(parte.function_call.name)

            # Capturar respuesta final del agente
            if (
                hasattr(parte, "text")
                and parte.text
                and evento.author == agent.root_agent.name
            ):
                respuesta_final = parte.text

    if herramientas_usadas:
        print(f"  [Herramientas usadas: {', '.join(herramientas_usadas)}]")

    return respuesta_final


async def conversacion_interactiva():
    """Ejecuta una conversación interactiva en la terminal.

    El usuario puede escribir mensajes y ver las respuestas del agente.
    Escribe 'salir' para terminar la conversación.
    """
    # Configurar el runner
    runner = Runner(
        agent=agent.root_agent,
        app_name="demo_programatico",
        session_service=InMemorySessionService(),
    )

    config = {
        "app_name": "demo_programatico",
        "user_id": "usuario_demo",
        "session_id": "sesion_interactiva_001",
    }

    print("\n" + "="*60)
    print("DEMO: Agente de Soporte TechStore")
    print("Escribe 'salir' para terminar")
    print("="*60 + "\n")

    turno = 0

    while True:
        turno += 1

        # Solicitar input del usuario
        try:
            mensaje = input(f"[Turno {turno}] Tú: ").strip()
        except KeyboardInterrupt:
            print("\n\nConversación terminada por el usuario.")
            break

        if mensaje.lower() in ["salir", "exit", "quit", "q"]:
            print("¡Hasta pronto!")
            break

        if not mensaje:
            print("Por favor escribe un mensaje.")
            turno -= 1  # No contar turnos vacíos
            continue

        # Ejecutar el agente
        print("Agente: ", end="", flush=True)
        respuesta = await ejecutar_un_turno(runner, mensaje, **config)
        print(respuesta)
        print()
